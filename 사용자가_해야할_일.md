# 사용자가 해야 할 일

목표는 최종 사용자에게 `PythonOnlineTetris.exe` 파일 하나만 전달하는 것입니다.

개발자/배포자인 당신이 한 번만 해야 하는 일은 아래 순서입니다.

## 전체 순서

1. Fly.io 계정 만들기
2. Fly CLI 설치하기
3. Fly.io에 로그인하기
4. 서버 앱 이름 정하기
5. 서버를 비용 최소화 설정으로 배포하기
6. 서버 주소를 클라이언트 설정에 넣기
7. 온라인 접속 테스트하기
8. exe 파일 만들기
9. exe 파일만 다른 사람에게 전달하기

## 1. Fly.io 계정 만들기

Fly.io 사이트에서 계정을 만듭니다.

```text
https://fly.io
```

Fly.io는 새 계정에서 카드 등록을 요구할 수 있습니다. 이 프로젝트는 비용이 최대한 적게 나오도록 설정하지만, Fly.io 정책상 완전 무료를 보장할 수는 없습니다.

비용을 줄이기 위해 이 프로젝트는 아래 방식으로 운영합니다.

- 서버를 계속 켜두지 않음
- 접속이 없으면 서버 자동 정지
- 누가 접속하면 서버 자동 시작
- 데이터베이스, 볼륨, 고정 IPv4, Redis 같은 추가 유료 리소스 사용 안 함

## 2. Fly CLI 설치하기

Fly.io 서버를 배포하려면 `flyctl`이라는 명령어 프로그램이 필요합니다.

설치 문서:

```text
https://fly.io/docs/flyctl/install/
```

설치가 끝나면 PowerShell을 새로 열고 아래 명령을 실행합니다.

```powershell
fly version
```

버전 정보가 나오면 설치가 된 것입니다.

## 3. Fly.io 로그인하기

PowerShell에서 아래 명령을 실행합니다.

```powershell
fly auth login
```

브라우저가 열리면 Fly.io 계정으로 로그인합니다.

로그인이 끝난 뒤 PowerShell에서 아래 명령으로 확인합니다.

```powershell
fly auth whoami
```

내 계정 이메일이나 사용자 정보가 나오면 정상입니다.

## 4. 서버 앱 이름 정하기

Fly.io 서버 주소는 앱 이름으로 정해집니다.

예를 들어 앱 이름이 아래와 같으면:

```text
python-online-tetris-server
```

서버 주소는 아래처럼 됩니다.

```text
https://python-online-tetris-server.fly.dev
```

현재 프로젝트는 이미 이 이름으로 설정되어 있습니다.

```toml
app = "python-online-tetris-server"
```

이 이름이 이미 다른 사람이 쓰고 있으면 배포가 실패합니다. 그때는 [fly.toml](fly.toml)의 `app` 값을 다른 이름으로 바꿔야 합니다.

예:

```toml
app = "my-gtris-server"
```

앱 이름을 바꾸면 나중에 `client/assets/config/server.json`의 주소도 같이 바꿔야 합니다.

## 5. 비용 최소화 설정 확인하기

[fly.toml](fly.toml)에 아래 설정이 들어 있어야 합니다.

```toml
[http_service]
  internal_port = 8000
  force_https = true
  auto_stop_machines = "stop"
  auto_start_machines = true
  min_machines_running = 0
```

각 설정의 뜻은 아래와 같습니다.

- `auto_stop_machines = "stop"`: 접속이 없으면 서버를 끕니다.
- `auto_start_machines = true`: 누가 접속하면 서버를 다시 켭니다.
- `min_machines_running = 0`: 항상 켜둘 서버 수를 0개로 합니다.

이 설정이 비용을 가장 줄이는 핵심입니다.

단점도 있습니다.

- 오랫동안 아무도 접속하지 않으면 서버가 꺼집니다.
- 그 뒤 첫 번째 접속자는 서버가 켜지는 동안 몇 초 기다릴 수 있습니다.
- 친구와 같이 테스트할 때 첫 접속이 한 번 느릴 수 있습니다.

그래도 5달러 미만으로 쓰고 싶다면 이 방식이 맞습니다.

## 6. 서버 배포하기

처음 한 번은 아래 명령을 실행합니다.

```powershell
fly launch
```

중간에 물어보는 것이 나오면 아래 기준으로 선택합니다.

- Postgres 만들기: No
- Redis 만들기: No
- Volume 만들기: No
- 유료 부가 기능: 만들지 않기
- Region: 가까운 곳 선택

한국에서 테스트하면 Tokyo/Narita 계열 지역이 가까운 편입니다. 현재 설정은 `nrt`입니다.

```toml
primary_region = "nrt"
```

이미 앱이 만들어져 있거나 `fly launch`가 끝났다면 서버를 배포합니다.

```powershell
fly deploy
```

배포가 끝나면 서버 상태를 확인합니다.

```powershell
fly status
```

## 7. 서버 주소 확인하기

브라우저에서 아래 주소를 열어봅니다.

```text
https://python-online-tetris-server.fly.dev
```

앱 이름을 바꿨다면 주소도 바뀝니다.

예:

```text
https://my-gtris-server.fly.dev
```

처음 접속할 때는 서버가 켜지는 데 몇 초 걸릴 수 있습니다.

## 8. 클라이언트 서버 주소 설정하기

exe 안에 서버 주소가 들어가야 최종 사용자가 주소를 입력하지 않아도 됩니다.

수정할 파일:

```text
client/assets/config/server.json
```

현재 설정:

```json
{
  "server_url": "https://python-online-tetris-server.fly.dev",
  "server_ws_url": "wss://python-online-tetris-server.fly.dev"
}
```

만약 Fly.io 앱 이름을 바꿨다면 이 파일도 바꿉니다.

예:

```json
{
  "server_url": "https://my-gtris-server.fly.dev",
  "server_ws_url": "wss://my-gtris-server.fly.dev"
}
```

`https` 주소는 일반 API용이고, `wss` 주소는 온라인 게임 실시간 연결용입니다. 둘 다 같은 앱 이름을 써야 합니다.

## 9. 개발용 실행 테스트하기

exe를 만들기 전에 먼저 개발 환경에서 실행해 봅니다.

```powershell
python -m client.main
```

확인할 것:

- 게임 창이 열리는지
- Online 메뉴로 들어갈 수 있는지
- 서버 연결 상태가 정상인지
- 방 목록이 보이는지
- `C` 키로 방 생성이 되는지
- 다른 클라이언트에서 방에 입장할 수 있는지
- 두 플레이어가 `R` 키로 READY 했을 때 온라인 매치 화면으로 넘어가는지
- 한 명이 게임오버되면 승패가 표시되는지
- 매치 종료 후 방으로 돌아갈 수 있는지

혼자 테스트할 때는 PowerShell 창을 2개 열고 각각 아래 명령을 실행하면 됩니다.

```powershell
python -m client.main
```

두 창을 각각 P1, P2처럼 사용해서 테스트합니다.

## 10. exe 빌드하기

필요한 패키지를 설치합니다.

```powershell
pip install -r requirements.txt
```

exe를 만듭니다.

```powershell
.\scripts\build_client_exe.ps1
```

성공하면 아래 파일이 생깁니다.

```text
dist/PythonOnlineTetris.exe
```

최종 사용자에게는 이 exe 파일만 전달하면 됩니다.

## 11. exe 단독 실행 테스트하기

빌드가 끝났으면 바로 전달하지 말고 먼저 직접 확인합니다.

1. `dist/PythonOnlineTetris.exe`를 실행합니다.
2. Online 메뉴에 들어갑니다.
3. 방 생성이 되는지 확인합니다.
4. 가능하면 다른 폴더로 exe를 복사해서 실행해 봅니다.
5. 다른 컴퓨터에서도 실행해 봅니다.

다른 폴더나 다른 컴퓨터에서 실행했을 때도 온라인 접속이 되면 배포 준비가 된 것입니다.

## 13. 서버를 강제로 꺼서 비용 줄이기

테스트가 끝났고 당분간 아무도 게임을 하지 않는다면 서버를 직접 꺼도 됩니다.

```powershell
fly machine list
```

Machine ID를 확인한 뒤:

```powershell
fly machine stop 머신ID
```

하지만 `auto_start_machines = true`라서 누가 다시 접속하면 서버가 다시 켜질 수 있습니다.

앱 자체를 완전히 지우고 싶을 때만 아래 명령을 사용합니다.

```powershell
fly apps destroy 앱이름
```

이 명령은 서버 앱을 삭제하므로 정말 필요할 때만 사용합니다.

## 14. 최종 체크리스트

배포 전에 아래 항목을 모두 확인합니다.

- Fly.io 로그인이 되어 있음

- [fly.toml](fly.toml)에 `auto_stop_machines = "stop"`이 있음
- [fly.toml](fly.toml)에 `min_machines_running = 0`이 있음
- Postgres, Redis, Volume을 만들지 않았음
- `fly deploy`가 성공함
- 서버 주소가 브라우저에서 열림
- [client/assets/config/server.json](client/assets/config/server.json)의 주소가 실제 서버 주소와 같음
- `python -m client.main`으로 온라인 테스트가 됨
- `.\scripts\build_client_exe.ps1` 빌드가 성공함
- `dist/PythonOnlineTetris.exe`가 실행됨
- exe만 실행해도 온라인 접속이 됨

## 15. 자주 생기는 문제

### 앱 이름이 이미 사용 중이라고 나오는 경우

[fly.toml](fly.toml)의 `app` 값을 다른 이름으로 바꿉니다.

그 다음 [client/assets/config/server.json](client/assets/config/server.json)의 주소도 같은 이름으로 바꿉니다.

### 첫 접속이 느린 경우

정상입니다.

비용을 줄이기 위해 서버가 꺼져 있다가 첫 접속 때 다시 켜지는 방식입니다.

### 온라인 연결이 안 되는 경우

먼저 서버가 배포되어 있는지 확인합니다.

```powershell
fly status
```

그 다음 서버 주소가 맞는지 확인합니다.

```text
client/assets/config/server.json
```

`server_url`은 `https://...fly.dev`여야 하고, `server_ws_url`은 `wss://...fly.dev`여야 합니다.

### `websocket disconnected`가 뜨고 상대 화면에 내 움직임이 안 보이는 경우

온라인 매치에서 상대 화면은 WebSocket으로 동기화됩니다. 그래서 WebSocket이 끊기면 내 움직임이 상대 화면에 보이지 않습니다.

먼저 서버 주소가 맞는지 확인합니다.

```text
client/assets/config/server.json
```

예:

```json
{
  "server_url": "https://내앱이름.fly.dev",
  "server_ws_url": "wss://내앱이름.fly.dev"
}
```

주의할 점:

- `server_url`은 `https://`로 시작해야 합니다.
- `server_ws_url`은 `wss://`로 시작해야 합니다.
- 두 주소의 앱 이름이 같아야 합니다.

그 다음 서버를 다시 배포합니다.

```powershell
cd "C:\Users\master\Desktop\파이썬 프로젝트"
fly deploy --config .\fly.toml
```

클라이언트를 exe로 실행 중이었다면 exe도 다시 만들어야 합니다.

```powershell
.\scripts\build_client_exe.ps1
```

개발 실행으로 확인할 때는 PowerShell 창 2개를 열고 각각 실행합니다.

```powershell
python -m client.main
```

한쪽에서 방을 만들고, 다른 한쪽에서 들어간 뒤 두 플레이어가 `R`로 READY 합니다.

첫 접속 때만 잠깐 느리거나 연결 메시지가 뜨는 것은 Fly.io 서버가 자동으로 켜지는 중일 수 있습니다. 하지만 매치가 시작된 뒤에도 계속 `websocket disconnected`가 뜨면 서버 주소 또는 배포 상태를 다시 확인해야 합니다.

### 비용이 예상보다 많이 나오는 경우

Fly.io 대시보드에서 추가 리소스가 생겼는지 확인합니다.

특히 아래 항목을 확인합니다.

- Postgres
- Volume
- Dedicated IPv4
- Static Egress IP
- 여러 개의 Machine
- 서버가 계속 Running 상태인지

필요 없는 리소스는 삭제해야 비용이 줄어듭니다.

### `app does not have a Dockerfile or buildpacks configured` 에러가 나오는 경우

이 에러는 대부분 `fly deploy`를 프로젝트 루트 폴더가 아닌 곳에서 실행했을 때 생깁니다.

반드시 아래 폴더에서 실행해야 합니다.

```text
C:\Users\master\Desktop\파이썬 프로젝트
```

PowerShell에서 아래처럼 이동합니다.

```powershell
cd "C:\Users\master\Desktop\파이썬 프로젝트"
```

그 다음 `Dockerfile`과 `fly.toml`이 보이는지 확인합니다.

```powershell
dir Dockerfile
dir fly.toml
```

둘 다 보이면 아래 명령으로 다시 배포합니다.

```powershell
fly deploy --config .\fly.toml
```

그래도 같은 에러가 나오면 Dockerfile 위치를 직접 지정해서 배포합니다.

```powershell
fly deploy --config .\fly.toml --dockerfile .\Dockerfile
```

주의할 점:

- `client` 폴더 안에서 실행하면 안 됩니다.
- `server` 폴더 안에서 실행하면 안 됩니다.
- GitHub에서 다운로드한 zip 파일 안의 다른 하위 폴더에서 실행하면 안 됩니다.
- `Dockerfile`, `fly.toml`, `server`, `shared` 폴더가 한 화면에 같이 보이는 위치에서 실행해야 합니다.

### `Error: app not found` 에러가 나오는 경우

이 에러는 [fly.toml](fly.toml)에 적힌 앱 이름이 내 Fly.io 계정에 아직 없다는 뜻입니다.

현재 설정은 아래 이름입니다.

```toml
app = "python-online-tetris-server"
```

이 앱을 내 계정에 새로 만들려면 프로젝트 루트에서 아래 명령을 실행합니다.

```powershell
cd "C:\Users\master\Desktop\파이썬 프로젝트"
fly apps create python-online-tetris-server
fly deploy --config .\fly.toml
```

만약 `python-online-tetris-server`라는 이름을 다른 사람이 이미 쓰고 있다면 앱 생성이 실패합니다. 그럴 때는 나만의 이름으로 바꿔야 합니다.

예:

```toml
app = "my-gtris-server"
```

그 다음 아래 명령을 실행합니다.

```powershell
fly apps create my-gtris-server
fly deploy --config .\fly.toml
```

앱 이름을 바꿨다면 [client/assets/config/server.json](client/assets/config/server.json)도 같은 이름으로 바꿔야 합니다.

예:

```json
{
  "server_url": "https://my-gtris-server.fly.dev",
  "server_ws_url": "wss://my-gtris-server.fly.dev"
}
```

앱이 만들어졌는지 확인하려면 아래 명령을 실행합니다.

```powershell
fly apps list
```

## 16. GitHub에 프로젝트 올리기

이 프로젝트를 GitHub에 올리려면 아래 순서대로 하면 됩니다.

먼저 GitHub 사이트에서 새 저장소를 만듭니다.

```text
https://github.com/new
```

추천 설정:

- Repository name: `python-online-tetris` 같은 이름
- Public 또는 Private 중 원하는 것 선택
- Add a README file: 체크하지 않기
- Add .gitignore: 체크하지 않기
- Choose a license: 지금은 선택하지 않아도 됨

이 프로젝트에는 이미 `.gitignore` 파일이 있으므로 GitHub에서 새로 만들지 않는 것이 좋습니다.

GitHub 저장소를 만든 뒤, 이 프로젝트 폴더에서 PowerShell을 열고 아래 명령을 실행합니다.

```powershell
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/내아이디/저장소이름.git
git push -u origin main
```

예를 들어 GitHub 아이디가 `myname`이고 저장소 이름이 `python-online-tetris`라면:

```powershell
git remote add origin https://github.com/myname/python-online-tetris.git
git push -u origin main
```

처음 `git push`를 할 때 GitHub 로그인을 요구할 수 있습니다.

비밀번호가 아니라 GitHub Personal Access Token을 요구할 수 있습니다. 그 경우 GitHub에서 토큰을 만들어 사용해야 합니다.

토큰 생성 위치:

```text
https://github.com/settings/tokens
```

권한은 개인 프로젝트를 올리는 용도라면 보통 `repo` 권한이 필요합니다.

### 이미 `git init`을 한 적이 있는 경우

아래 명령으로 상태를 확인합니다.

```powershell
git status
```

원격 저장소가 연결되어 있는지도 확인합니다.

```powershell
git remote -v
```

원격 저장소가 없다면 아래 명령으로 추가합니다.

```powershell
git remote add origin https://github.com/내아이디/저장소이름.git
```

이미 잘못된 원격 저장소가 들어가 있다면 아래처럼 바꿉니다.

```powershell
git remote set-url origin https://github.com/내아이디/저장소이름.git
```

그 다음 올립니다.

```powershell
git push -u origin main
```

### GitHub에 올리면 안 되는 파일

아래 파일이나 폴더는 GitHub에 올리지 않는 것이 좋습니다.

- `.env`
- `__pycache__/`
- `.pytest_cache/`
- `dist/`
- `build/`
- `*.exe`
- `*.log`
- 저작권이 불확실한 음악, 효과음, 이미지, 폰트 파일

이 프로젝트의 `.gitignore`가 위 파일들을 제외하도록 설정되어 있습니다.

### GitHub에 올린 뒤 수정사항을 다시 올리는 방법

파일을 수정한 뒤에는 아래 세 명령을 반복하면 됩니다.

```powershell
git add .
git commit -m "수정 내용 설명"
git push
```

예:

```powershell
git add .
git commit -m "Update deployment guide"
git push
```
