# 온라인 테트리스 프로젝트 구조 및 계획

## 목표

`pygame-ce`와 `FastAPI`를 사용해 파이썬으로만 온라인 테트리스 게임을 구현한다. 게임 흐름은 `Intro`, `Menu`, `Single`, `Online`, `Options` 씬으로 나누고, 이후 배경음악, 효과음, 이펙트, 애니메이션을 확장할 수 있는 구조로 만든다.

## 구조 설명

`client/`는 `pygame-ce`로 실행되는 게임 클라이언트다. 화면 전환, 입력 처리, 테트리스 규칙, UI, 네트워크 통신, 사운드, 이펙트를 모두 포함한다.

`server/`는 `FastAPI` 서버다. 방 생성, 방 목록, 레디 상태, 온라인 매치 시작, WebSocket 연결 관리를 담당한다. Fly.io 배포를 고려해 서버 진입점은 `server/main.py`로 단순하게 유지한다.

`shared/`는 클라이언트와 서버가 같이 쓰는 메시지 규격을 둔다. 온라인 게임에서는 프로토콜이 자주 바뀌므로 `shared/protocol.py`, `shared/schemas.py`에 이벤트 이름과 데이터 구조를 모아두는 것이 좋다.

`client/audio/`와 `client/effects/`는 처음부터 비워두더라도 별도 모듈로 분리한다. 나중에 배경음악, 효과음, 라인 클리어 이펙트, 하드드랍 충격, 화면 흔들림, 씬 전환 효과를 추가할 때 기존 게임 로직을 덜 건드리게 된다.

## 주요 씬 계획

### 1. Intro

- 로고 또는 시작 화면 표시
- 일정 시간 후 메뉴로 이동
- Enter 입력 시 즉시 메뉴로 이동
- 추후 배경음악 페이드인, 화면 전환 효과 추가

### 2. Menu

- `Single`, `Online`, `Options`, `Exit` 항목 제공
- Tab 또는 방향키로 이동
- Enter로 선택
- Esc로 이전 화면 또는 종료 확인

### 3. Single

- 오프라인 테트리스 플레이
- 조작키: 좌우 이동, 회전, 180도 회전, 소프트드랍, 하드드랍, 홀드
- 게임오버 후 R로 다시하기
- 온라인 구현 전 핵심 테트리스 규칙을 검증하는 기준 모드로 사용

### 4. Online Lobby

- 방 목록 조회
- 방 제목, 비밀번호 사용 여부 표시
- 방 생성 화면 제공
- 선택한 방에 입장

### 5. Online Room

- 입장한 플레이어 목록 표시
- 각 플레이어의 레디 상태 표시
- 둘 다 레디하면 게임 시작
- R 키로 준비 상태 전환

### 6. Online Game

- 내 보드와 상대 보드 표시
- 서버와 WebSocket으로 입력 또는 상태 동기화
- 라인 클리어, 공격 줄, 승패 처리
- 연결 끊김, 상대 퇴장, 재시작 처리

### 7. Options

- 조작키 변경
- 음악 on/off
- 효과음 on/off
- 볼륨 설정
- 전체화면 여부 등 확장 가능

## 구현 단계

### 1단계: 프로젝트 뼈대

- `client/main.py`에서 pygame 창 실행
- 씬 매니저 구현
- Intro, Menu, Options 빈 화면 연결
- 기본 입력 시스템 구현

### 2단계: 싱글 플레이 핵심

- 보드, 블록, 충돌 판정 구현
- 7-bag 랜덤 생성기 구현
- 이동, 회전, 하드드랍, 소프트드랍, 홀드 구현
- 라인 클리어와 게임오버 구현
- 점수, 레벨, 다음 블록 UI 추가

### 3단계: UI와 설정 저장

- 버튼, 리스트, 텍스트, 입력 박스 컴포넌트 구현
- 조작키 변경 화면 구현
- 음악/효과음 on/off 설정 저장
- 설정 파일을 JSON으로 저장

### 4단계: 서버 기본 기능

- FastAPI 서버 생성
- 방 생성, 방 목록 조회, 방 입장 API 구현
- WebSocket 연결 구현
- 플레이어 레디 상태 동기화
- 두 명이 레디하면 매치 시작 이벤트 전송

### 5단계: 온라인 게임

- 클라이언트 Online Lobby, Online Room 연결
- WebSocket 메시지 프로토콜 고정
- 온라인 게임 시작, 입력 전송, 상태 동기화 구현
- 승패, 나가기, 연결 끊김 처리

### 6단계: 사운드와 이펙트

- `AudioManager`로 음악과 효과음 재생 통합
- 씬별 배경음악 설정
- 이동, 회전, 하드드랍, 홀드, 라인 클리어, 게임오버 효과음 추가
- `EffectManager`로 파티클, 화면 흔들림, 전환 효과 관리
- 옵션의 음악/효과음 설정과 연결

### 7단계: 배포와 테스트

- 서버 테스트 작성
- 클라이언트 핵심 규칙 테스트 작성
- Fly.io 배포 설정 추가
- 로컬 서버와 배포 서버 접속 설정 분리
- 간단한 스모크 테스트 작성

## 온라인 프로토콜 초안

```text
room.create
room.list
room.join
room.leave
room.ready
match.start
match.input
match.state
match.garbage
match.result
error
```

초기에는 서버가 방과 매치 상태를 관리하고, 클라이언트는 입력과 화면 표시를 담당하는 구조가 적합하다. 이후 동기화 문제가 생기면 서버 권한을 강화하거나, 입력 기반 동기화 방식으로 조정한다.

## 사운드/이펙트 확장 기준

- 게임 로직에서 직접 `pygame.mixer`를 호출하지 않는다.
- 게임 로직은 `audio.play_sfx("hard_drop")`처럼 이벤트 이름만 전달한다.
- 이펙트도 `effects.spawn("line_clear", position=...)`처럼 요청만 보낸다.
- 자산 파일명은 기능 기준으로 둔다. 예: `hard_drop.wav`, `line_clear.wav`, `menu_theme.ogg`
- 옵션 설정은 `AudioManager`, `EffectManager`가 읽어서 적용한다.

## 우선순위

1. 싱글 플레이가 재미있고 안정적으로 동작하게 만든다.
2. 온라인 로비와 방 시스템을 붙인다.
3. 온라인 매치 동기화를 구현한다.
4. 사운드와 이펙트로 완성도를 올린다.
5. 서버 배포와 테스트를 정리한다.
